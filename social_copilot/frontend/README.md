# Social Copilot (AI 社交军师)

一款基于本地 LLM 的智能聊天分析与回复建议桌面应用。

## 🧱 当前前端骨架（2026-02）

当前版本已切到双窗口骨架，便于后续和 Python Visual Monitor 联调：

- `Main Console`：矩形主控台窗口（设置与状态承载）
- `Assistant Bubble`：圆形常驻悬浮窗口（素材未接入时使用占位视觉）

> 说明：助手头像素材可后补，不影响骨架开发与联调。

## 📋 前置要求

在运行应用之前，请确保你的电脑已安装以下软件：

### 1. Node.js (v18+)
```bash
# 检查是否已安装
node --version

# 如未安装，使用 Homebrew 安装
brew install node
```

### 2. Ollama (本地 LLM 运行时)
```bash
# 安装 Ollama
brew install ollama

# 启动 Ollama 服务
ollama serve

# 在新终端窗口下载 qwen3 模型（约 5GB）
ollama pull qwen3:8b
```

> ⚠️ **重要**: Ollama 服务必须在后台运行，应用才能正常工作！

## 🚀 快速开始

### 步骤 1: 安装依赖
```bash
cd social_copilot/frontend
npm install
```

### 步骤 2: 确认 Ollama 运行中
```bash
# 检查 Ollama 是否正常运行
curl http://localhost:11434/api/tags
```
如果返回 JSON 数据，说明 Ollama 正常运行。

### 步骤 3: 启动应用
```bash
npm run dev
```

应用将自动打开两个窗口：

- 主控台窗口（矩形）
- 圆形助手窗口（常驻悬浮）

## 🎯 功能测试指南

### 测试场景 1: 基本聊天分析

1. **打开应用** - 确认主界面显示正常，包含：
   - 左侧：聊天输入区域
   - 右侧：建议面板
   - 顶部：联系人选择器

2. **输入聊天记录** - 在输入框中粘贴以下示例对话：
```
2024-01-15 10:30:00 小明: 最近工作怎么样？
2024-01-15 10:31:00 我: 还行吧，有点忙
2024-01-15 10:32:00 小明: 周末有空吗？想约你吃饭
```

3. **点击提交** - 观察：
   - [ ] 是否显示加载状态
   - [ ] 是否在 5 秒内返回 3 条回复建议
   - [ ] 每条建议是否包含回复内容和策略说明

### 测试场景 2: 复制建议到剪贴板

1. 生成建议后，**点击任意一条建议卡片**
2. 观察：
   - [ ] 是否显示"已复制"提示
   - [ ] 打开微信/备忘录，按 Cmd+V 粘贴，确认内容正确

### 测试场景 3: 用户画像设置

1. **点击设置图标**（齿轮按钮）
2. 修改以下信息：
   - 性别
   - 职业
   - 语气风格（正式/轻松/幽默）
3. **保存设置**
4. 重新提交聊天记录，观察：
   - [ ] 生成的建议是否符合你设置的语气风格

### 测试场景 4: 联系人管理

1. 在聊天记录中使用不同的联系人名称
2. 观察：
   - [ ] 联系人选择器是否自动添加新联系人
   - [ ] 切换联系人时是否加载对应的画像信息

### 测试场景 5: 导入聊天文件

1. 创建一个 `.txt` 文件，内容格式如下：
```
2024-01-15 14:00:00 朋友A: 你好啊
2024-01-15 14:01:00 我: 你好！
2024-01-15 14:02:00 朋友A: 最近在忙什么呢？
```

2. **点击导入按钮**，选择该文件
3. 观察：
   - [ ] 文件内容是否正确解析
   - [ ] 是否生成回复建议

### 测试场景 6: 错误处理

1. **关闭 Ollama 服务**（在终端按 Ctrl+C）
2. 重新提交聊天记录
3. 观察：
   - [ ] 是否显示清晰的错误提示
   - [ ] 是否提供故障排除步骤

4. **重新启动 Ollama**：
```bash
ollama serve
```

### 测试场景 7: 空输入验证

1. 不输入任何内容，直接点击提交
2. 观察：
   - [ ] 是否阻止提交
   - [ ] 是否显示验证提示

## 📁 数据存储位置

应用数据存储在 `~/SocialCopilot/` 目录：

```
~/SocialCopilot/
├── user_profile.json      # 用户画像
└── contacts/              # 联系人画像
    ├── 小明.json
    └── 朋友A.json
```

你可以直接查看这些文件来验证数据是否正确保存。

## 🔧 常见问题

### Q: 应用启动后显示 Ollama 连接错误
A: 确保 Ollama 服务正在运行：
```bash
ollama serve
```

### Q: 建议生成很慢
A: 首次调用模型需要加载到内存，可能需要 10-20 秒。后续调用会快很多。

### Q: 如何重置所有数据？
A: 删除数据目录：
```bash
rm -rf ~/SocialCopilot
```
